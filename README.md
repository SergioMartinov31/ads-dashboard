#  Авито AI-ассистент для продавцов

Веб-приложение — личный кабинет продавца с интегрированным AI-ассистентом. Помогает улучшать описания объявлений, анализировать характеристики товара и предлагать рыночную цену.

**Основной стек:** React 19, TypeScript, Vite, Redux Toolkit, Mantine UI.

---

## 📁 Структура проекта

- `client/` — фронтенд на React + Vite
- `server/` — mock-backend API с данными объявлений
- `llm-server/` — AI-прокси для общения с языковой моделью

### Ключевые папки фронтенда

- `src/app/` — провайдеры `store` и `router`
- `src/entities/ad/` — типы и API для объявлений
- `src/features/` — поиск, AI, редактирование объявлений
- `src/pages/` — страницы списка, просмотра, редактирования
- `src/widgets/` — повторно используемые компоненты интерфейса

## 🚀 Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone <url-вашего-репозитория>
cd <папка-проекта>
```

### 2. Установите зависимости

```bash
cd client
npm install

cd ../server
npm install

cd ../llm-server
npm install
```

### 3. Настройте окружение

#### `llm-server/.env`

Создайте файл `llm-server/.env` и добавьте ваш ключ OpenRouter или другой совместимый ключ:

```env
API_KEY=ваш-ключ-openrouter
```

> Важно: ключ не должен попадать в публичный репозиторий.

#### `client/.env`

Файл `client/.env` не обязателен. Значения по умолчанию настроены в коде:

```env
VITE_API_URL=http://localhost:8080
VITE_AI_URL=http://localhost:3001
```

## ▶️ Запуск проекта

Запускайте серверы в отдельных терминалах из корня проекта.

### 1) Запуск backend API

```bash
cd server
PORT=8080 npm start
```

Backend будет доступен на:

```text
http://localhost:8080
```

### 2) Запуск LLM-прокси

```bash
cd llm-server
npm run dev
```

Proxy будет доступен на:

```text
http://localhost:3001
```

### 3) Запуск frontend

```bash
cd client
npm run dev
```

Откройте приложение по адресу, который покажет Vite (обычно `http://localhost:5173`).

## 📌 Что делает приложение

### 1. Страница списка объявлений `/ads`

- Поиск по названию объявления
- Фильтрация по категориям: авто, недвижимость, электроника
- Переключатель «Только требующие доработок»
- Сортировка по дате и названию
- Пагинация по 10 объявлений на страницу
- Карточки с заглушкой изображения, категорией, ценой и бейджем «Требует доработки»

### 2. Страница просмотра объявления `/ads/:id`

- Полная информация: название, цена, дата публикации
- Характеристики в зависимости от категории
- Блок с перечислением незаполненных полей
- Кнопка «Редактировать» для перехода на страницу редактирования

### 3. Страница редактирования объявления `/ads/:id/edit`

- Обязательные поля: категория, название, цена
- Динамические характеристики в зависимости от категории
- Описание с подсчетом текста
- Кнопки AI:
  - «Придумать описание» / «Улучшить описание»
  - «Узнать рыночную цену»
- Автосохранение черновика в `localStorage`
- Сохранение через `PUT /items/:id`
- Отмена — возврат на страницу просмотра без сохранения

## 🤖 AI-интеграция

AI-функции реализованы через отдельный прокси-сервер `llm-server`.

- фронтенд отправляет данные на `http://localhost:3001`
- прокси формирует промпты и отправляет запрос на LLM
- в текущей реализации используется OpenRouter через модель `gpt-3.5-turbo`

### Поддерживаемые AI-функции

- `description` — генерация или улучшение описания товара
- `price` — оценка рыночной цены с диапазонами

### Пример формата ответа для цены

```text
Средняя цена:
115 000 – 135 000 ₽ — отличное состояние.
90 000 – 110 000 ₽ — идеал.
70 000 – 85 000 ₽ — срочно / с дефектами.
```

## 🧠 Принятые решения

| Вопрос | Решение | Обоснование |
|---|---|---|
| Сохранение черновиков | `localStorage` с ключом `ad-draft-{id}` | Простое решение без дополнительного backend и защита от перезагрузки страницы |
| Управление состоянием | Redux Toolkit / RTK Query | Удобное кэширование, загрузка данных и обработка ошибок без лишнего шаблона |
| UI | Mantine | Быстрая стилизация, готовые компоненты и поддержка TypeScript |
| AI-прокси | Отдельный Node.js сервер | Изоляция API-ключа, замена провайдера без изменения фронтенда |

## 🧪 Возможные проблемы при запуске

### AI-кнопки возвращают ошибку

- Проверьте `llm-server/.env`
- Убедитесь, что ключ `API_KEY` указан
- Перезапустите `llm-server` и проверьте подключение
